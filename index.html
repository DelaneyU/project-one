ul Back to top

The html ul element stands for unordered list. Unordered lists are bullet point lists because they do not have a specific order. The are no attributes for this element, if you want to edit it you'll have to use css.

<ul> <li> input text</li> </ul>

ol Back to top

The html ol element represents an ordered list. ordered lists are numbered unlike unordered lists, ordered lists have a a valued importance to them. Just like unorderd lists you can only style ol's in css.

<ol> <li> input text</li> </ol>

li Back to top

The html li element means list. List html is used in ordered and unordered lists.

  • value - Specifies the value of the element.

<li> input text </li>

a Back to top

The <a> tag defines a hyperlink, and is used to link from page to page. When using the <a> you must use the href attribute if not, other attributes cannot be represented.

  • href - Specifies the URL of the page the link goes to.
  • hreflang - Specifies the language of the linked document.
  • download - Specifies that the target will be downloaded when a user clicks on the hyperlink.
  • media - Specifies what media/device the linked document is optimized for.
  • rel - Specifie the relationship between the current document and the linked document.
  • target - Specifies the target for where to open the linked document or where to submit the form.
  • type - Specifies the type of element.

<a href="url"> title of link </a>

img Back to top

The html img element represents an image. Its used to place an image in your content.

  • alt - Specifies an alternate text when the origianl element fails to display.
  • height - Specifies the height of an image.
  • width - Specifies the width of the element.
  • src - Specifies the URL of the media file.
  • ismap - Specifies an image as a serverside image-map.
  • onabort - Script to be run on abort.
  • onerror - Script to be run when error occurs.
  • onload - Script to be run when the element if finished loading.
  • sizes - Specifies the size of the linked resource.
  • srcset - Specifies the URL of the image to use in different situations.
  • usemap - Specifies an image as a clientside image-map

<img src="value" alt="input text">

<!DOCTYPE html> <html>

head Back to top

The head descirbes the page and links you to other resources intoducing content to your page. Elements that can go in a head include; title, style, base, link, meta, script, and nonscript.

<head>input text </head>

link Back to top

The html link element tag defines a link between a document and an external resource. In addition to linking to external style sheets. Its and empty element only containing attributes. Links are defined with an <a> tag.

  • href - Specifies the location of the linked document.
  • hreflang - Specifies the language of the text in the linked document.
  • media - Specifies on what device the linked document will be displayed.
  • rel - Required. Specifies the relationship bewteen the current document.
  • sizes - Specifies the size of the linked resource. Only for rel="icon".

<a href="url"> link text </a>

title Back to top

The title tag is required in html documents, defining the title of the document. It defines the title in the browser toolbar and displayed a title for the page in search-engine results. You cannot have more than one title in a document.

<title> title text </title>

body Back to top

The body is where you place all of your content; heading, paragraphs, images, etc. basically anything you want to be seen.

  • onafterprint - Script to be run after the document is printed.
  • onbeforeprint - Script to be run before the document is printed.
  • onbeforeunload - Script to be run when the document is about to be unloaded.
    • onerror - Script to be run when an error occurs.
  • onhashchange - Script to be run when there has been changes to the anchor part of the a URL.
  • onload - Script to be run when the element is finsihed loading.
  • onoffline - Script to be run when the browser starts to work offline.
  • ononline - Script to be run when the browser starts to work online.
  • onpagehide- Script to be run when a user naviagtes away from a page.
  • onpageshow - Script to be run when a user navigates to a page.
  • onpopstate - Script to be run when the window's history changes.
  • onresize - Script to be run when the browser window is being resized.
  • onstorage - Script to be run when a Web Storage area is updated.
  • onunload - Script to be run when a page has unloaded (or the browser window has been closed).

<body> Content </body>

span Back to top

The html span tag is used to group inline-elements in a document. The tag provides no visual change by itself and provides a way to add a hook to a part of a text or a part of a document. When text is hooked in a span element, you can style it with CSS, or manipulate it with JavaScript.

<span></span>

div Back to top

The div tag defines a division or a section in an html document. It also is used as a container for other html elements to style them with CSS.

<div>content</div>

header Back to top

The header element represents a container for introductory content or a set of navigational links. Usually contains one or more heading elements, a logo or icon. You can have several header elements in a document.

<header>content</header>

nav Back to top

The nav tag defines a set of navigation links with in the navigation bar. All links of the document do not belong in the nav tag, only the major block of navigation links.

<nav> <a href="url"> link text </a> </nav>

footer Back to top

The footer tag defines a footer for a document or section. It should contain information about its containing element. The footer typically contains; authorship information, copyright information, contact information, sitemap, back to top links, and related documents. You can have several footer elements in a document.

<footer>content</footer>

main Back to top

The main tag specifies the main content of a document. The content inside the main element should be unique to the document. It should not contain any content that is repeated across documents such as sidebars, navigation links, copyright information, site logos and search forms. There must not be more than one main element in a document. It must not be a descendant of an article, aside, footer, header, or nav element.

<main>content</main>

section Back to top

The section tag defines sections in a document, such as chapters, headers, footers, or any other section of the document.

<section>content</section>

aside Back to top

The aside tag defines some content aside from the content it is placed in. Aside content should be related to the surrounding content.

<aside>content</aside>
